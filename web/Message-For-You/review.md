# 배운 점

- flask에서 세션 만들때 서명할 키가 필요하다.

- flask 세션 구조
  - `Payload` (데이터): 실제 내용이 저장되며, 내용이 많으면 `.`으로 시작하며 `zlib` 압축이 된다.
  - `Timestamp` (시간): 세션이 생성되거나 마지막으로 수정된 시간 정보로 세션이 만료 여부를 확인한다.
  - `Signature` (서명): 세션을 만들 때 사용한 키값을 이용해 만든 해시값으로 변조 방지를 위해 사용됨.

- `zlib` 압축을 사용하는 경우
  세션 데이터(JSON 직렬화 결과)가 100바이트 이상이되면 `zlib` 압축한다.

  압축이 되면 쿠키의 첫 번째 마디가 `.`로 시작한다.

  브라우저는 보통 4KB 쿠키 크기 제한이 있기 때문에, 압축해서 사용한다.

# 풀이 과정 기록

세션을 받아온 값이 `jwt`와 유사하게 생겨서 혹시나 `jwt` 디코딩 해보았지만 별다른 수확이 없었다.

flask의 세션 처리방식을 몰라 찾아본 결과 `jwt`와 유사하게 3종류로 나누어 base64 인코딩한다는 것을 알게됐고, `base64` 디코딩을 실시했으나 이상한 값이 확인되었다.

flask에서 base64 인코딩 할 때 패딩 문자를 버리는 것과 용량이 크면 `zlib` 압축을 실행한다는 점, url safe 방식으로 인코딩한다는 것을 확인한 후에, 역으로 접근하여 flag를 획득했다.
