# 배운 점

- `source`를 `.`로 쓸 수 있다는 점
- `source`는 현재 쉘에서 파일을 실행한다.
- 실행 대상이 스크립트가 아닐 경우, 각 라인을 명령어로 해석한다. 이때 존재하지 않는 명령어가 실행되면 에러 메시지가 출력된다.

# 풀이 과정 기록

한 글자를 써서 `/app/flag.txt`를 출력하는 문제인데, 어떤 한 글자를 써야되는 지 감이 안왔다.

한 글자로 의심되는 여러 특수기호를 눌러보다 `.` 우연하게 맞췄다.

찾아보니 `source` 명령어와 같은데 `.`로 쓸 수 있다는 점.
현재 쉘 세션에서 셸 스크립트 파일 실행하는 용도인데, flag를 읽을 때 쉘스크립트 파일이 아닌 파일을 읽게되면서 에러로 스크립트로 읽을 수 없는 파일의 라인이 출력되서 첫줄에 입력된 flag값이 노출되는 경우로 문제를 해결하는 것이였다.
